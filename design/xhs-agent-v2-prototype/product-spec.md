# XHS Agent V2 原型 · Product Spec
> 采集日期：2026-05-14
> 来源：当前仓库前端页面、后端 agent/tool 能力、用户连续反馈
> 资产完整度：本地产品，无外部品牌资产依赖

## 设计目标
- 面向普通用户：低门槛地完成「对话 → 成稿 → 配图 → 诊断 → 发布」。
- 面向专业写手：保留标题/正文/标签/分镜/评分/历史诊断等精细操控。
- 面向管理员：把模型配置、生图测试、静态图片公网可访问测试、用户角色管理放在清晰闭环中。
- 面向 Agent：让工具调用有边界、有进度、有耗时、有结果可写回。

## 风格方向
选择「小红书编辑部指挥舱」：奶油纸感底色 + 中国红强调 + 墨色高密信息 + 手机预览固定锚点。避免泛 SaaS 冷淡感，强调创作者场景和内容资产感。

## 页面覆盖
1. 登录页
2. 创作/Agent 首页
3. 笔记库
4. 笔记编辑详情
5. 诊断评分页
6. 模板库
7. 数据分析
8. 任务中心
9. 设置/管理员

## 关键交互提案
- 左侧统一导航 + 当前任务上下文条：任何页面都能看到当前关联笔记。
- Agent 消息中显示：工具名称、进度、耗时、图文分镜、失败原因与重试方案。
- 图片队列首图即封面，封面/内容图边界弱化，支持拖拽排序。
- 诊断历史可复看；再次诊断时出现「查看历史 / 重新诊断」。
- 设置页增加真实生图测试与静态图片公网测试，部署风险前置暴露。
