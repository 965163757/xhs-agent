Design Directions

设计假设

这是一个「高密度 AI 内容生产工作台」，不是营销官网。用户的核心任务是持续创作、审稿、改图、追踪后台任务，因此界面需要优先表达可控性、上下文和恢复能力。

位置四问

叙事角色：工作台总览；观众距离：笔记本 60–90cm；视觉温度：温暖但专业；容量估算：每个主屏至少呈现 3 个真实产品差异化信息点。

A · 推荐Josef Müller-Brockmann 式信息建筑

用明确网格、紧凑密度和工具链状态表达「Agent 可控」。适合作为当前系统主方向，最利于承载后台任务、模型队列、诊断链路和管理员视角。

  • 高密度三栏工作台
  • 少装饰，重状态和 trace
  • 红色只用于关键动作
BKenya Hara 式纸感留白

把「写笔记」的编辑体验变得更像创作纸张，适合面向普通用户降低工具压迫感。但后台任务和模型配置页面需要保留更强的信息密度。

  • 奶白纸感背景
  • 编辑区更轻、更安静
  • 图片队列像素材纸夹
CCharles Eames 式创作控制室

强调「创作流水线」和可视化决策，适合未来做团队版、运营版。视觉更有动态感，但若过度使用会让个人创作者觉得负担稍重。

  • 任务、评分、图片并列监控
  • 流程节点显性化
  • 适合多人协作扩展
本次原型采用 A 为主，吸收 B 的纸感背景和 C 的任务可观察性。

理由：系统目前最需要解决的是「Agent 到底在做什么、失败在哪里、用户能不能恢复」；视觉表达优先服务稳定感和控制感，而不是单纯漂亮。

Auth

登录 / 注册

简洁鉴权入口，登录成功后回到创作台；错误信息直接展示，保留注册开关能力。

  • Tab 切换登录/注册
  • 表单 busy 状态
  • 鉴权后 ProtectedRoute

未登录用户会被重定向到 /login
