# XHS Agent Prototype v2 · 重做设计简报
> 日期：2026-05-15  
> 状态：上一版已废弃，本版从零重做  
> 目标：不是说明书式原型，而是一个可以感知 Agent 正在工作的创作工作台。

## 为什么重做
上一版的问题：
- 太像「功能说明面板」，产品现场感不足。
- 动效被放成一个说明页面，而不是贯穿系统行为。
- 页面之间仍然是传统后台结构，缺少创作产品的气质。

## 新方向
### 「Editorial Command Studio」
把系统看成一个小红书内容团队的智能编辑室：
- 左侧是工作室导航与用户/管理员上下文。
- 中间是当前生产线：对话、笔记、图片、诊断、任务可以在同一条流线上切换。
- 右侧是「Live Context」：当前笔记、图片、模型、任务、评分始终可见。

## 哲学元素
1. **信息建筑**：复杂状态必须排队、分层、可追踪。
2. **编辑室纸感**：标题、正文、标签、版本像稿纸，而不是后台表单。
3. **动态生产线**：Agent 行为以流线、脉冲、任务卡落位表达，不做装饰动画。
4. **小红书内容感**：只在封面预览、标签、发布状态中出现，不污染后台能力页。

## 动效策略
- 页面切换：像换工作台，不像换 PPT。
- Agent 工具：从对话气泡旁生成，落到任务流。
- 图片生成：占位图逐步显影。
- fallback：模型队列有接管路径，不静默。
- 诊断：分项评分按顺序点亮。

## 交付范围
覆盖：登录、创作对话、笔记列表、编辑器、诊断、模板、数据、任务、设置、管理员视角、图片编辑弹窗、命令面板。
