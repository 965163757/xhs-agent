## 设计方向
- **产品定位**：面向小红书内容生产的 Agent 工作台，覆盖创作、笔记管理、图片生成/编辑、诊断、模板、数据、后台任务和模型配置。
- **视觉关键词**：高密度、轻量红书感、Agent 可观察性、编辑器工作台、可恢复后台任务。
- **颜色资产**：主红 `#FF2442` / 暖橙 `#FF7A00` / 纸感背景 `#F8F8F6` / 墨色 `#1A1A1A` / 成功绿 `#16A34A` / 风险橙 `#D97706`。
- **字体资产**：`Inter`, `PingFang SC`, `HarmonyOS Sans SC`, `Noto Sans SC`, system-ui。
- **核心识别**：沿用当前 AppBar 「红」字渐变方标，避免引入虚构品牌素材。

## Agent 能力在 UI 中的呈现
- **对话上下文**：当前笔记是默认上下文但不强绑定；可在首页或编辑页选择任意笔记；管理员可按用户查看所有对话和笔记。
- **工具调用感知**：消息卡片展示工具名、参数、结果、图片尝试链路、耗时、成功/失败、重试建议。
- **图片上下文**：上传图片显示缩略图；历史图片保留路径；涉及视觉任务时附带图片；编辑成功后在聊天框展示结果图并支持放大/触发编辑。
- **后台任务**：聊天和诊断任务可刷新/关闭页面后继续，通过任务中心与 SSE 重连恢复。
- **图片生成/编辑**：图片模型候选队列支持 URL 能力、quality 能力、8 分钟预算、失败切备用、链路可视化。

## 原型覆盖策略
本原型采用单页高保真 HTML 工作台，左侧页面导航 + 中部对应页面 mock + 右侧功能清单。每个页面都不是空壳，保留当前系统关键交互：历史侧栏、命令面板、图片编辑弹窗、模型队列、诊断后台进度、管理员分组、任务 trace。
